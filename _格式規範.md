# 知識檔（.md）格式規範

> 給 AI 看的：每次產生／編輯 hegrn-kb 的知識 .md 檔，一律照這套做，不用再問 Ivan。
> 目標：在 LINE / Telegram 上「好閱讀」，因為 bot 會把這份 md 餵給 LLM 再回覆。

## 鐵則

1. 🚫 **絕對不用 markdown 表格**（`| ... |`）。表格在 LINE/TG 會爛成「· · ·」一串。改用「emoji 條列」呈現同樣的資訊。
2. 😀 **適時加 emoji**：每個區塊標題、每個重點開頭放一個相關 emoji，幫助掃讀。原則是「一個重點一個 emoji」，不要整句塞滿、也不要硬加到不相關處。
3. 📏 **短句、短段**：一個概念一行或一小段，避免長到要捲半天的段落。
4. ➖ **用 `---` 分隔大段落**，讓主題之間有呼吸感。
5. 💬 **白話優先**：能讓國小生聽懂才是真懂；先給比喻或觀念，再落到「明天就能做的一件事」。
6. 🪧 **檔案開頭埋一句提示**給 bot：例如「回答時請用條列＋emoji 呈現，不要用表格」，引導輸出格式。

## 表格 → 條列 的轉法（範例）

不要這樣（會爛掉）：
```
| 級別 | 類型 | 結果 |
|---|---|---|
| 4 | 豐盛交換 | 轉介、續報 |
```

改成這樣：
```
🏆 第 4 級・豐盛交換
拿到遠超過價值的東西 → 結果：主動轉介、續報、口碑擴散。
```

## 其他沿用的 SOP（搬家後版本）

- 檔名一律英文（中文檔名 raw 會 404）。
- 新主題：英文 .md 丟上 hegrn-kb repo → 編輯 catalog.md 加一行「主題名|關鍵字|網址」→ 等 2–5 分鐘快取 → 打「知識 …」測。
- 想讓「不打知識前綴」也自動命中，要把該主題關鍵字加進 LINE+TG 兩支 CEO 派工器的【派工目標：知識】區塊。
